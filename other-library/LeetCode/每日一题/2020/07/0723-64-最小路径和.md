所以，为了解决这个问题，势必要进行优化：

```js
/**
 * @param {number[][]} grid
 * @return {number}
 */
const minPathSum = (grid) => {
  if (!grid.length) {
    return 0;
  }
  const M = grid.length;
  const N = grid[0].length;
  for (let i = 0; i < M; i++) {
    for (let j = 0; j < N; j++) {
      // 从 [0, 0] 触发
      if (i === 0 && j === 0) {
        continue;
      }
      // 如果上左都存在，取两者最小
      if (grid[i - 1] && grid[i - 1][j] !== undefined && grid[i][j - 1] !== undefined) {
        grid[i][j] += Math.min(grid[i - 1][j], grid[i][j - 1]);
      } else if (grid[i - 1] && grid[i - 1][j] !== undefined) { // 如果只存在上
        grid[i][j] += grid[i - 1][j];
      } else if (grid[i][j - 1] !== undefined) { // 如果只存在左
        grid[i][j] += grid[i][j - 1];
      }
    }
  }
  return grid[M - 1][N - 1];
};
```

因为广度搜索要排出的机器人太多了，走的路线连 JavaScript 都存储不过来。

所以只能将每次结果最优存储起来，最终找出剩下的结果 `grid[M - 1][N - 1]`。
